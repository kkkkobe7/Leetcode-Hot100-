# 621. 任务调度器

解题思路拆解

**1. 寻找瓶颈：谁决定了最长时间？**
*   假设任务 `A` 出现了最多次（记为 `ma` 次），且冷却时间为 `n`。
*   为了让时间最短，我们应该尽可能先把 `A` 安排开。由于两个 `A` 之间必须隔 `n` 个位置，我们可以把安排看成一个“架子”：
    *   `A` _ _ (n个空格) `A` _ _ (n个空格) ... `A`
*   这个架子被分成了 `ma - 1` 个完整的块，每个块的长度是 `n + 1`（1个A加上n个空格）。
*   **初步计算大小**：`(n + 1) * (ma - 1)`。

**2. 填充末尾：处理并列最多的任务**
*   在最后一个 `A` 之后，可能还有一些任务要放。
*   如果还有其他任务（比如 `B`）也出现了 `ma` 次，那么它会紧跟在每一个 `A` 后面，包括最后一个 `A`。
*   **更新计算**：`(n + 1) * (ma - 1) + tot`。其中 `tot` 是出现次数同样为 `ma` 的任务种类数。

**3. 容纳剩余任务：如果架子放不下怎么办？**
*   如果我们的任务种类非常多，把上面的空格填满了还剩下一堆任务怎么办？
*   **结论**：在这种情况下，CPU 永远不需要“待命（Idle）”，它可以一直不停地工作。此时，最短时间就是任务的总数 `tasks.size()`。

**4. 总结公式**
*   最短时间 = `max(任务总数, 构造的架子大小)`。



简洁版代码：

```c++
class Solution {
public:
    int cnt[26];

    int leastInterval(vector<char>& tasks, int n) {
        for (auto& c: tasks) cnt[c - 'A'] ++;
        int ma = 0, tot = 0;
        for (int i = 0; i < 26; i ++) ma = max(ma, cnt[i]); // 找到最大单一任务数量
        for (int i = 0; i < 26; i ++) tot += int(ma == cnt[i]); // 同时为最大任务数量的任务个数

        return max((n + 1) * (ma - 1) + tot, (int)tasks.size());
    }
};
```

注释版代码：

```c++
class Solution {
public:
    int cnt[26]; // 统计 A-Z 每个字母出现的次数

    int leastInterval(vector<char>& tasks, int n) {
        // 1. 统计频率
        for (auto& c: tasks) cnt[c - 'A'] ++;

        // 2. 找到出现次数最多的任务次数 ma
        int ma = 0;
        for (int i = 0; i < 26; i ++) ma = max(ma, cnt[i]);

        // 3. 计算有多少种任务的出现次数正好等于 ma
        int tot = 0;
        for (int i = 0; i < 26; i ++) {
            if (cnt[i] == ma) tot++;
        }

        // 4. 利用公式计算最小时间
        // (ma - 1) 是指最高频任务形成的“间隔”数
        // (n + 1) 是每个间隔包含一个最高频任务和其后的冷却时间
        // tot 是最后一行（不再需要冷却）的任务数量
        int res = (n + 1) * (ma - 1) + tot;

        // 5. 最终结果：
        // 如果任务总数更多，说明格子被填满了，无需等待，结果就是 tasks.size()
        // 如果计算出的 res 更大，说明需要等待（Idle），结果就是 res
        return max(res, (int)tasks.size());
    }
};
```
