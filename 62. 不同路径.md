# 62. 不同路径

解题思路拆解

**1. 核心洞察：加法原理**
*   机器人只能**向右**或**向下**走。
*   反推：如果要到达坐标为 `(i, j)` 的格子，机器人只有两种可能的来源：
    1.  从**上面**的一格 `(i-1, j)` 走下来。
    2.  从**左边**的一格 `(i, j-1)` 走过来。
*   **结论**：到达 `(i, j)` 的路径总数 = 到达上方的路径数 + 到达左边的路径数。

**2. 定义状态 (State Definition)**
*   我们定义 `f[i][j]` 为：到达坐标 `(i, j)` 的不同路径总数。
*   为了方便处理边界，通常可以使用 **1-based indexing**（下标从 1 开始）。

**3. 状态转移方程 (Transition)**
*   公式：`f[i][j] = f[i-1][j] + f[i][j-1]`。
*   *注意：你在代码里写了 `max(f[i][j], ...)`，在数组初始值为 0 的情况下这和相加是一样的，但在 LeetCode 多组数据测试环境下，建议直接相加并确保数组清零。*

**4. 初始状态 (Base Case)**
*   起点：`f[1][1] = 1`（起始位置本身就有 1 种方案）。
*   第一行和第一列：虽然公式 `f[i][j] = f[i-1][j] + f[i][j-1]` 依然适用，但直观来看，由于只能向右或向下，第一行和第一列的每个格子都只有 **1** 种走法。



简洁版代码：

```c++
class Solution {
public:
    static const int N = 110;
    int f[N][N];
    
    int uniquePaths(int m, int n) {
        f[1][1] = 1;
        for (int i = 1; i <= m; i ++)
        {
            for (int j = 1; j<= n; j ++)
            {
                f[i][j] = max(f[i][j], f[i-1][j] + f[i][j-1]);
            }
        }
        return f[m][n];
    }
};
```

注释版代码：

```c++
class Solution {
public:
    // N 定义为 110，覆盖 m, n <= 100 的范围
    static const int N = 110;
    int f[N][N];
    
    int uniquePaths(int m, int n) {
        // 【面试重点】重置 DP 数组
        // 在 LeetCode 中，类成员变量可能会被多个测试用例复用
        // 如果不清空，上一个测试用例的结果会留在 f 里
        memset(f, 0, sizeof f);

        // 1. 初始化起点
        f[1][1] = 1;

        // 2. 嵌套循环遍历网格
        // i 代表行，j 代表列
        for (int i = 1; i <= m; i ++)
        {
            for (int j = 1; j <= n; j ++)
            {
                // 如果是起点，跳过（已经初始化为 1 了）
                if (i == 1 && j == 1) continue;

                // 3. 核心状态转移
                // 到达当前格子的路径数 = 上方格子路径数 + 左方格子路径数
                // 这里的 f[i-1][j] 和 f[i][j-1] 如果越界（下标为0），
                // 因为我们 memset 过了，它们的值是 0，计算也是正确的
                f[i][j] = f[i-1][j] + f[i][j-1];
            }
        }

        // 4. 返回右下角的结果
        return f[m][n];
    }
};
```
