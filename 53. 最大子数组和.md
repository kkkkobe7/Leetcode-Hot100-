# 53. 最大子数组和

解题思路拆解

**1. 核心矛盾：及时止损**
*   我们从左向右累加数组元素。
*   如果我们发现当前的累加和（`count`）已经变成了**负数**，那么它对于后面的序列来说，就是一个“累赘”。
*   **例子**：如果前面加起来是 `-2`，后面一个数是 `4`。
    *   继续带着前面的话：`-2 + 4 = 2`。
    *   直接舍弃前面，从 4 开始：结果是 `4`。
*   **结论**：如果当前的累加和小于等于 0，我们应该立即将其重置为 0，表示从下一个位置开始重新计算子数组。

**2. 为什么能处理全负数？**
*   如果数组里全是负数（比如 `[-2, -1, -3]`）：
    *   `result` 初始化为 `INT32_MIN`。
    *   遇到 `-2`，`count` 变成 `-2`，更新 `result` 为 `-2`。然后 `count` 归零。
    *   遇到 `-1`，`count` 变成 `-1`，更新 `result` 为 `-1`。然后 `count` 归零。
*   最终返回的结果就是数组中**最大的那个负数**。这符合“子数组最少包含一个元素”的要求。

**3. 动态规划的影子**
*   这其实是动态规划的空间优化版。
*   **状态定义**：`dp[i]` 是以第 `i` 个数字结尾的最大子数组和。
*   **转移方程**：`dp[i] = max(nums[i], dp[i-1] + nums[i])`。
*   你的 `count = 0` 并在 `count < 0` 时重置，本质上就是这个 `max` 操作。

简洁版代码：

```c++
class Solution {
public:
    // 如果不是全负数的话，最大子数组和一定是由正数开始、正数结束
    int maxSubArray(vector<int>& nums) {
        int result = INT32_MIN;
        int count = 0, n = nums.size();
        for (int i = 0; i < nums.size(); i++) {
            count += nums[i];
            if (count > result) result = count;
            if (count <= 0) count = 0; // 负数一定不在开头段
        }
        return result;
    }
};
```

注释版代码：

```c++
class Solution {
public:
    // 核心思想：贪心/动态规划。一旦前面的累加和为负，立即舍弃。
    int maxSubArray(vector<int>& nums) {
        int result = INT32_MIN;
        // count 用于存储当前的连续累加和
        int count = 0; 
        
        for (int i = 0; i < nums.size(); i++) {
            // 2. 累加当前元素
            count += nums[i];
            
            // 3. 更新全局最大值
            // 只要当前的累加和刷新了记录，就记下来
            if (count > result) {
                result = count;
            }
            
            // 4. 核心贪心逻辑：舍弃负数贡献
            // 如果 count 变成了负数，说明它对后面的求和只有副作用
            // 哪怕下一个数是 100，带着这个负数 count 也会让结果变小（比如 -2+100=98 < 100）
            // 所以我们将其清零，相当于“从下一个位置重新开始计算子数组”
            if (count <= 0) {
                count = 0; 
            }
        }
        
        return result;
    }
};
```
